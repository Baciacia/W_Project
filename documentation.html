1. Abstract

The purpose of this document is to present a detailed description of our web application designed to manage information related to a zoo.It will explain the purpose and features of the web application, the interfaces of the software, what the web application will do and the constraints under which it must operate.

2. Introduction

Furry Park is a web-application for a zoo, and is meant to be user-friendly and eye candy.In our vision, it should allow the users to browse through animal information like an encyclopedia and it should be a virtual guide for the zoo.

Also, we want large groups to be able to book their visit and a guide if they want.Individuals can buy tickets online to avoid waiting in line.To expand our zoo, we implemented a donation page for people to help us. For ease of access, users can create accounts, so they don't have to re-enter their personal data every time.

3. Overall Description

Furry Park is based on HTML (the base of the site), CSS (for design elements) and javascript (for dynamic and interactive content).

3.1 Product Functions

  • Account creation
  • Exporting and importing animal information
  • Search filters
  • Donations platform
  • Booking platform

3.2 User Classes and Characteristics

Visitor:

  • Account creation
  • Search filters
  • Donations platform
  • Booking platform

Admin:

  • Everything the visitor can do
  • Exporting and importing animal information

3.3 Operating Environment

The software will run on anything that supports a browser (such as phones running on Android and computers running on Windows and Linux).

4. External Interface Requirements

4.1 User Interfaces

The menu bar of the website is present on all the other pages of the site and it directs the user to some other key parts of the site.

The wepage is responsive so that it can adapt to any screen size. 

                
                @media screen and (max-width:800px) {
                    .container-v{
                        width: 28rem;
                        margin-bottom: 10rem;
                    }
                
                    .body-v{
                        height: 60rem;
                    }
                }

There will be an error displayed if any field of the forms is filled incorrectly.

4.2 Hardware Interfaces

The minimum hardware requirements of the web-app are a Pentium 4 CPU or equivalent, 256 megabytes of RAM and 100MB disk space.

4.3 Software Interfaces

The minimum requirements for the operating system are: minimum Windows XP SP2, OS X 10.5.6, Ubuntu 10.04, Debian 6, OpenSuse 11.3, Fedora Linux 14 and any recent operating system for smartphones.Also, any recent version of any browser that supports HTML5.

4.4 Communications Interfaces

The web aoolication dose not require an internet connection or any other communication interface

5. System Features

This section demonstrates the website's most prominent features and explains how they can be used and the results they will give back to the user.

5.1 Import/export animal pages

If a user is logged in as an admin they can use the import/export button on the animal page. Otherwise there will be an error displayed. The admin can add an animal page by importing a file with the necessary information. They can also export an animal page that is already on the website.

The available input/output file format is json.

5.2 Filters

On the animal page there is a search bar based on keywords/tags. The user will recieve a list of animal pages based on the applied filters.

On the bottom right side of the screen there is the reset button which removes the selected filters.
